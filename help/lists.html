A list is an ordered sequence of values. The values can be any data type, and they don't all have to be the same. Write a list as square brackets, surrounding zero or more values separated by commas. The following are all valid lists:

["uno", "dos", "tres"]
[8, 6, 7, 5, 3, 0, 9]
[1, 3.14157, "spam", ["a", "b", "c"]]
[]

It's legal (and quite common) to create an empty list with just a pair of square brackets, as in the last example above.

Lists support most of the same operators as strings:

+list concatenation
*replication (i.e. repeat the list some number of times)
/division (get some fraction of a list)
==, !=comparison (for equality)
[i]get/set item i (first item is 0)
[i:j]get sublist ("slice") from i up to j

The indexes i and j above are nifty: if negative, they count from the end of the list. If nonnegative, they count forward, starting at zero. Also, with the slice operator, you can omit either or both values, and they default to the start and end of the list. Examples, all supposing that x = ["a", 42, 3.14, 7, "hike"]:

x[0]"a" (first item)
x[1]42 (second item)
x[-1]"hike" (last item)
x[-2]7 (next-to-last item)
x[1:-1][42, 3.14, 7] (everything from the second up to the last item)
x[1:][42, 3.14, 7, "hike"] (everything from the second item to the end)
x[:-1][42, 3.14, 7, "hike"] (everything up to the last item)

Slicing and indexing of lists (and strings) is incredibly powerful once you get the hang of it.

Unlike strings, lists are mutable: you can change them in place by assigning to an element, or by using any of the mutator methods (which we'll document soon).
