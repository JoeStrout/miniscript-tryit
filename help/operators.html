MiniScript supports the following operators.

=assignment
function()function definition
orlogical "or" (with short-circuit evaluation)
andlogical "and" (with short-circuit evaluation)
notlogical "not"
isatests whether left-hand value is of right-hand type
==, !=, >, >=, <, <=value comparison
+addition (numbers); concatenation (strings, lists, maps)
-subtraction
*, /, %multiplication/division (numbers); replication (strings, lists); mod (numbers)
-unary minus (numbers only)
newreturns a new map with __isa set to right-hand value
@address-of (get reference to function without invoking it)
^exponentiation (numbers)
.map lookup/call
[]index/slice; list definition
{}map definition

The operators above are listed in reverse-precedence order, that is, ones near the bottom of the list take precedence over ones towards the top.

Note that the logical operators (and, or, and not) work with any data type; strings, lists, and maps are considered true if they are nonempty. null is always considered false.

Numbers are considered true if nonzero, at least for branching purposes, but the "standard" truth values (always generated by or, and, and not) are 1 for true and 0 for false.
